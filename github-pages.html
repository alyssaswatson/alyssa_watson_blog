Moving Github Pages to a Google Domains

I recently moved my personal website and domain from Wordpress to GitHub pages, and found the process to be more confusing than I originally anticipated, so I hope this helps you with moving to GitHub pages even if you aren’t moving from Wordpress or using Google Domains. I bought a domain through Wordpress. If you did this, you likely need to transfer your domain to a hosting service that will be more flexible to your needs. This is relatively straightforward, but does take several days to actually take effect. To move a Github Pages site to a domain you own, you need to:

    • Create a new repository on GitHub, and add all the necessary code to make it look and act the way you’d like. If you already have your site setup, just continue on to the next step. https://pages.github.com/
      Your site will be published at https://yourusername.github.io/what_you_named_the_repository/. You can also find this under the settings tab in your repository.
      On the settings tab, there is a section under “GitHub Pages” called “Custom domain.” Put the domain name without the ‘www’ in that box and press ‘save.’
      There are some instructions on GitHub for setting up an apex domain. I found them to be a little overwhelming when first looking at them, but the only section we really need is pictured below. https://help.github.com/articles/setting-up-an-apex-domain/
      On your Google Domains page, click on the “DNS” symbol next to the domain you are trying to move your page to.
      Fill out the “Custom resource records” section at the bottom with the two IP addresses from the GitHub section screen as I have done.
      Be patient. While mine starting working within a matter of minutes, this could theoretically take up to 24 hours.

    It really is not a hard process once you know what information you need and where it goes. I hope if you had any trouble, these steps help you setup your custom domain on GitHub pages using Google Domains!
